WIFI AVANCÉ

September 6, 2023 · 3 min read
PANCHOO Ashvin
Etudiant en Réseaux et Télécoms
NANY Emerick
Etudiant en Réseaux et Télécoms

Pages 2-16 : Compréhension essentielle

  • Conversion Fréquence-Espace : Lambda est égal à C divisé par f. Cela signifie que nous convertissons des données de fréquence en données spatiales.

  • Attribution des Bandes de Fréquence : L'Union internationale des télécommunications, via son Règlement des radiocommunications, alloue différentes bandes de fréquences aux services de radiocommunication. Ces attributions sont décidées lors des conférences mondiales des radiocommunications.

  • Réseaux :

    • PAN (Personal Area Network) : A une portée d'environ quelques mètres autour de l'utilisateur.
    • HAN (Home Area Network) : S'étend jusqu'à 10 mètres autour d'une station relais.
    • LAN (Local Area Network) : Peut aller jusqu'à quelques centaines de mètres. Si sans fil, c'est appelé WLAN (Wireless LAN).
    • WAN (Wide Area Network) : Couvre une grande distance, allant de quelques centaines à des milliers de kilomètres.

Pages 18-42 : Fondamentaux de la Transmission

  • Analyse Fréquentielle : Si un phénomène est difficile à comprendre dans le domaine temporel, il faut l'examiner dans le domaine fréquentiel. Pour passer du temporel au fréquentiel, on utilise la Transformée de Fourier, et vice versa avec sa version inverse.

  • Échantillonnage : Cela signifie prendre des valeurs d'un signal à des intervalles spécifiques. Selon le Théorème De Shannon, le taux d'échantillonnage ( Fe ) doit être supérieur à 2 fois la fréquence maximale du signal ( Fmax ).

  • Bande Passante : C'est la gamme de fréquences dans laquelle les signaux sont transmis efficacement. Il est essentiel que le spectre du signal transmis soit compris dans cette bande.

  • Valence : Représente le nombre d'états possibles d'un signal. Pour un signal binaire, elle est de 2 (soit 0 soit 1). Par exemple, la capacité maximale de transmission ( D max ) est de 6000 bit/s pour une valence de 2.

  • Bruit : Il s'agit de toute perturbation non désirée qui affecte la transmission. Le rapport signal/bruit (SNR) mesure la puissance du signal par rapport à celle du bruit. Un SNR élevé indique un signal puissant.

  • Atténuation : C'est la diminution de l'énergie du signal lors de sa propagation. Elle est souvent plus prononcée à des fréquences élevées.

  • Distorsion Temporelle : Différentes parties d'un signal peuvent se déplacer à différentes vitesses, causant une distorsion.

  • Sources de Bruit : Il peut provenir de l'émetteur, du milieu de transmission ou de perturbations extérieures. Le bruit thermique, causé par l'agitation des électrons, est courant.

  • Modulation : C'est la modification d'un message pour sa transmission. Elle vise à transposer le signal dans une bande de fréquence adaptée, à protéger contre le bruit et à permettre des transmissions simultanées.

  • Techniques de Transmission :

    • Étalement de Spectre : Il s'agit de disperser l'énergie d'un signal sur une large bande de fréquences.
    • CDMA : Permet à plusieurs utilisateurs d'émettre sur la même fréquence grâce à des codes uniques.
    • OFDMA : Divise le canal principal en sous-canaux, chacun étant modulé par une fréquence différente pour une utilisation optimale de la bande.

TP2-2 Installation du SDWAN ZTNET

September 4, 2023 · 3 min read
PANCHOO Ashvin
Etudiant en Réseaux et Télécoms
NANY Emerick
Etudiant en Réseaux et Télécoms

Présentation​

ZTNET est une solution SD-WAN qui permet aux organisations de déployer facilement des réseaux définis par logiciel. Cette documentation vise à guider les utilisateurs à travers le processus d'installation de ZTNET sur une machine virtuelle (VM) Ubuntu en utilisant Docker, une plateforme de conteneurisation. En suivant ces étapes, les utilisateurs pourront mettre en place une instance ZTNET et accéder à son interface web pour gérer leurs réseaux.

Prérequis​

  • Une VM Ubuntu Server.
  • Docker installé.

Étapes d'installation​

a. Créez le fichier docker-compose​

  1. Créez un fichier appelé docker-compose.yml.
  2. Copiez et collez le contenu fourni ci-dessus dans ce fichier.
version: "3.1"
services:
  postgres:
    image: postgres:15.2-alpine
    container_name: postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: ztnet
    volumes:
      - postgres-data:/var/lib/postgresql/data
    networks:
      - app-network

  zerotier:
    image: zyclonite/zerotier:1.10.6
    hostname: zerotier
    container_name: zerotier
    restart: unless-stopped
    volumes:
      - zerotier:/var/lib/zerotier-one
    cap_add:
      - NET_ADMIN
      - SYS_ADMIN
    devices:
      - /dev/net/tun:/dev/net/tun
    networks:
      - app-network
    ports:
      - "9993:9993/udp"
    environment:
      - ZT_OVERRIDE_LOCAL_CONF=true
      - ZT_ALLOW_MANAGEMENT_FROM=172.31.255.0/29

  ztnet:
    image: sinamics/ztnet:latest
    container_name: ztnet
    working_dir: /app
    volumes:
      - zerotier:/var/lib/zerotier-one
    restart: unless-stopped
    ports:
      - 3000:3000
    environment:
      POSTGRES_HOST: postgres
      POSTGRES_PORT: 5432
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: ztnet
      NEXTAUTH_URL: "http://localhost:3000"
      NEXTAUTH_SECRET: "random_secret"
    networks:
      - app-network
    links:
      - postgres
    depends_on:
      - postgres
      - zerotier
volumes:
  zerotier:
  postgres-data:

networks:
  app-network:
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 172.31.255.0/29

b. Lancer ZTNET​

  1. Ouvrez un terminal.
  2. Naviguez vers le répertoire où se trouve le fichier docker-compose.yml.
  3. Exécutez la commande:
    docker-compose up -d
  4. Une fois l'action terminée, ouvrez un navigateur et visitez http://localhost:3000 pour accéder à l'interface web ZTNET.

c. Première connexion​

  • Lors de la première inscription sur l'interface ZTNET, le premier utilisateur enregistré obtiendra automatiquement les privilèges d'administrateur.

Points à noter​

  • En tant qu'administrateur, vous avez des capacités uniques non disponibles pour les utilisateurs normaux, telles que la capacité de voir tous les comptes enregistrés sur le contrôleur.
  • Les administrateurs ne peuvent pas interagir ou modifier directement les réseaux des autres utilisateurs.

Variables d'environnement​

  1. Vous pouvez personnaliser plusieurs variables d'environnement incluses dans le fichier docker-compose.yml selon vos besoins.
  2. Les variables comme POSTGRES_HOST, POSTGRES_PORT, etc., sont associées à la base de données PostgreSQL et à NextAuth pour l'authentification.
  3. Pour modifier ces valeurs, mettez simplement à jour la variable d'environnement correspondante dans le fichier docker-compose.yml.
  4. Il est conseillé de ne pas modifier les variables d'environnement systémiques utilisées par le service ZeroTier, sauf si vous savez ce que vous faites.

Conclusion​

En suivant ces étapes, vous aurez configuré et lancé ZTNET sur votre VM Ubuntu. Assurez-vous de bien gérer les privilèges administratifs et de comprendre l'impact des modifications apportées aux variables d'environnement pour garantir le fonctionnement optimal de votre installation.
